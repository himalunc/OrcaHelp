Data Chain Set-up

 

In order to take data, you must define a data chain in the ORCA configuration to tell ORCA the run conditions, what hardware to readout, and what to do with the data. A typical data chain consists of a Run Control, Data Taker, Data Monitor, and Disk File connected together thus:

Run Control

Data Storage

Data Readout control

Fan Out

That would generally be the minimum data chain, but as an option you might need to broadcast your data to a remote machine for backup or for analysis. In that case you would increase the number of outputs in the Fan Out object to three and attach a Data Broadcaster. In addition, there is a RunNotes object that can be used to insert arbitrary label/value pairs into the data header.
